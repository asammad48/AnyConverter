How to Query a CSV File with SQL

  1. Upload your CSV — Drop or select a CSV file with a header row. It loads into an in-browser SQLite database as a table named data.
  2. Write SQL — Query with full SQL: SELECT, WHERE, GROUP BY, ORDER BY, LIMIT, JOINs, aggregates. Use Ctrl+Enter to run quickly.
  3. Explore and export — Sort results by clicking column headers. Navigate pages. Export the full result set as CSV or JSON.

Features

  • Full SQLite SQL — SELECT, WHERE, GROUP BY, ORDER BY, aggregates
  • Paginated results — 50 rows per page
  • Sortable columns — click headers to sort asc/desc
  • Export results as CSV or JSON
  • Query history — last 5 queries, click to restore
  • 100% browser-based — powered by SQL.js (SQLite WASM)

Frequently Asked Questions

What SQL syntax is supported? +
The tool uses SQL.js, a WebAssembly build of SQLite. It supports SELECT, WHERE, GROUP BY, ORDER BY, LIMIT, OFFSET, JOINs, subqueries, and aggregates like COUNT, SUM, AVG, MIN, and MAX. Most SQLite functions work too.
What is the table name for my CSV? +
Your CSV is loaded into a table called data. Column names come from the first row (header row) of your CSV. If a column name has spaces, wrap it in double quotes: SELECT "first name" FROM data.
How large a file can I query? +
There is no hard limit — the browser's available memory is the constraint. Files up to 50MB typically work well. Very large files may take a moment to load into the in-memory SQLite database.
Is my data sent to a server? +
No. All processing runs locally in your browser. SQL.js is a WASM build of SQLite that runs entirely client-side. Your CSV data and query results never leave your device.
Can I run multiple queries on the same file? +
Yes. Once a CSV is loaded, you can run as many queries as you like. The query history keeps your last 5 queries for quick re-use. Upload a new file to replace the current dataset.
