Cómo Usar el Comparador de Texto

  1. Paso 1 — Pega tu texto original en el área de texto izquierda "Texto original".
  2. Paso 2 — Pega la versión modificada en el área de texto derecha "Texto modificado".
  3. Paso 3 — Haz clic en "Comparar" para ver las diferencias resaltadas línea por línea. Las líneas verdes fueron añadidas y las rojas eliminadas.

Características

  • Comparación línea a línea usando el algoritmo LCS (Subsecuencia Común más Larga)
  • Salida codificada por colores: verde para añadidas, rojo para eliminadas, gris para sin cambios
  • Estadísticas de resumen con el total de líneas añadidas, eliminadas y sin cambios
  • Funciona completamente en tu navegador — no se envía texto a ningún servidor
  • Compatible con cualquier texto plano: código, prosa, archivos de configuración y más
  • Salida diff desplazable para documentos largos
  • Botón Limpiar con un solo clic para reiniciar ambas entradas

Preguntas Frecuentes

¿Cómo funciona el algoritmo de comparación? +
La herramienta usa el algoritmo de Subsecuencia Común más Larga (LCS), la misma base que usa git diff. Encuentra el mayor conjunto de líneas que aparecen en ambos textos en el mismo orden y marca el resto como añadidas o eliminadas.
¿Se envía mi texto a un servidor? +
No. Toda la comparación ocurre dentro de tu navegador usando JavaScript. Tu texto nunca abandona tu dispositivo, lo que lo hace seguro para contenido sensible o confidencial.
¿Puedo comparar archivos de código? +
Sí. La herramienta funciona con cualquier texto plano, incluyendo código fuente en cualquier lenguaje, JSON, XML, YAML y archivos de configuración. Simplemente pega el contenido del archivo en ambas áreas de texto.
¿Qué significa el conteo de "Sin cambios"? +
Las líneas sin cambios son líneas que existen en ambos textos, original y modificado, en posiciones correspondientes. Un conteo alto de sin cambios significa que los dos textos son muy similares.
¿Existe un límite de tamaño para el texto que puedo comparar? +
No hay un límite impuesto, pero textos muy grandes (miles de líneas) pueden tardar un momento en procesarse. Para mejor rendimiento, compara textos de menos de unos miles de líneas.
