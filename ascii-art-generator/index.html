Frequently Asked Questions

What can I use ASCII art for? +
ASCII art is commonly used in code comments (to mark sections or file headers), terminal/console banners, README files, emails, text messages, and social media bios. It works anywhere plain text is displayed in a monospace font.
Why does ASCII art look misaligned in some places? +
ASCII art requires a monospace (fixed-width) font where all characters have the same width. In proportional fonts (like most email clients use), characters have different widths and the art appears distorted. Use a code editor or monospace font for best results.
What characters can I use as the fill character? +
Any single character works — # (default), * (asterisk), @ (at), . (period), X, 0, or even a Unicode block character. The character simply replaces the # symbols in the block letters.
Is there a character limit? +
Yes, input is limited to 20 characters. Longer text would produce very wide output that doesn't fit most screens. For longer text, generate it in multiple segments.
