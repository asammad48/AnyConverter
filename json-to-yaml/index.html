Frequently Asked Questions

What is YAML? +
YAML (YAML Ain't Markup Language) is a human-friendly data serialization format. It uses indentation and minimal punctuation, making it easier to read than JSON. It's widely used for configuration files (Docker, Kubernetes, GitHub Actions).
Is all JSON valid YAML? +
Yes — JSON is technically a subset of YAML 1.2. Any valid JSON document is also valid YAML. However, YAML supports additional features like comments and multi-line strings that JSON does not.
Does YAML support comments? +
Yes. YAML supports comments using the # character. JSON does not support comments at all. When converting YAML to JSON, comments are discarded.
What indentation does the converter use? +
The converter outputs YAML with 2-space indentation, which is the most common convention for YAML files in modern tooling.
Is my data sent to a server? +
No. This tool runs entirely in your browser using the js-yaml library. Your data is never uploaded to any server, making it safe for sensitive configuration data.
