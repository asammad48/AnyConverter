Frequently Asked Questions

What is Base64 encoding? +
Base64 is an encoding scheme that converts binary data to ASCII text using 64 printable characters. It's commonly used in email attachments, data URLs, and APIs.
Why does Base64 encoded text end with ==? +
The = characters are padding added when the input length is not a multiple of 3 bytes. One = means 1 byte of padding was added, two == means 2 bytes.
Does Base64 encrypt my data? +
No. Base64 is an encoding, not encryption. It can be trivially decoded. Do not use Base64 to secure sensitive data.
What is a data URL? +
A data URL embeds file content directly into a URL using the format data:[MIME type];base64,[encoded data]. This allows images and files to be used directly in HTML without a separate file.
Can I encode images to Base64? +
Yes. Use File Mode to drop or upload any image. You'll get both the raw Base64 string and the complete data URL ready to use in HTML or CSS.
