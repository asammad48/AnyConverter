Frequently Asked Questions

What is camelCase and when is it used? +
camelCase writes compound words with no spaces, capitalizing the first letter of each word except the first (e.g., myVariableName). It is widely used in JavaScript, Java, and Swift for naming variables and functions.
What is the difference between snake_case and kebab-case? +
snake_case uses underscores to separate words (e.g., my_variable_name) and is common in Python and database column names. kebab-case uses hyphens (e.g., my-variable-name) and is widely used in URLs, CSS class names, and HTML attributes.
Does Title Case capitalize every word? +
This tool capitalizes the first letter of every word in Title Case mode. Some style guides (APA, Chicago) have exceptions for short prepositions and articles, but this tool uses a simple "every word" approach suitable for most use cases.
Will the converter handle special characters and numbers? +
Yes, for UPPER, lower, Title, Sentence, and alternating case, special characters and numbers are preserved as-is. For programming cases like snake_case, kebab-case, and dot.case, non-alphanumeric characters are removed to ensure valid identifiers.
Can I use this to convert code variable names? +
Absolutely. Paste a phrase or existing variable name, click camelCase, PascalCase, snake_case, or kebab-case, and copy the result. This is great for renaming variables when switching between programming languages or style conventions.
