How to Convert CSV to SQL

  1. Input your CSV — Paste CSV data or upload a .csv file. Large files are processed in chunks so the page stays responsive.
  2. Configure column types — Review the auto-detected types (INT, VARCHAR, DATE, etc.) and adjust as needed while chunks load.
  3. Generate SQL — Set table name, choose dialect, and click Generate SQL. The page auto-scrolls to the output.

Frequently Asked Questions

What SQL dialects are supported? +
MySQL (backtick quoting, AUTO_INCREMENT), PostgreSQL (double-quote identifiers, SERIAL), and SQLite (INTEGER PRIMARY KEY AUTOINCREMENT).
How does chunked processing work? +
Large files are processed in 5 MB chunks using PapaParse streaming. The browser stays responsive and you can configure column types while remaining chunks load.
How does auto type detection work? +
The tool samples the first 50 rows of each column and checks if all values match integers, decimals, booleans, dates, or long text. It defaults to VARCHAR(255) for mixed types.
How are empty cells handled? +
Empty cells are converted to NULL in the SQL INSERT statements, which is the standard SQL representation of a missing value.
What is batch INSERT? +
Batch mode combines 500 rows into a single INSERT statement. This is much faster for large datasets when importing into a database.
