How to Use the Text Diff Checker

  1. Step 1 — Paste your original text into the left textarea labeled "Original Text".
  2. Step 2 — Paste the modified version into the right textarea labeled "Modified Text".
  3. Step 3 — Click "Compare" to see the differences highlighted line by line. Green lines were added, red lines were removed.

Features

  • Line-by-line diff using the LCS (Longest Common Subsequence) algorithm
  • Color-coded output: green for added, red for removed, gray for unchanged
  • Summary stats showing total lines added, removed, and unchanged
  • Works entirely in your browser — no text is sent to any server
  • Supports any plain text: code, prose, configuration files, and more
  • Scrollable diff output for long documents
  • One-click Clear button to reset both inputs

Frequently Asked Questions

How does the diff algorithm work? +
The tool uses the Longest Common Subsequence (LCS) algorithm, the same foundation used by tools like git diff. It finds the largest set of lines that appear in both texts in the same order, then marks everything else as added or removed.
Is my text sent to a server? +
No. All comparison happens entirely inside your browser using JavaScript. Your text never leaves your device, making it safe to use with sensitive or confidential content.
Can I compare code files? +
Yes. The tool works with any plain text, including source code in any language, JSON, XML, YAML, configuration files, and more. Simply paste the file contents into both text areas.
What does the "Unchanged" count mean? +
Unchanged lines are lines that exist in both the original and modified text at corresponding positions. A high unchanged count means the two texts are very similar.
Is there a size limit for the text I can compare? +
There is no enforced limit, but very large texts (thousands of lines) may take a moment to process due to the LCS algorithm's complexity. For best performance, compare texts under a few thousand lines.
